# Branch Contribution Types Guide

## Overview
Branch Contribution Types allow branch administrators to create custom contribution categories with specific allocation percentages for mission, area, district, and branch levels.

## Access
- **URL:** `/contributions/branch-types/`
- **Permissions:** Branch Admin, Mission Admin
- **Menu:** Contributions â†’ Branch Types

## Features

### 1. Create Contribution Type
- **Name:** Descriptive name for the contribution
- **Category:** Tithe, Offering, Special, Project, Welfare, Other
- **Description:** Optional details about the type
- **Allocation Percentages:** Define how funds are distributed

### 2. Allocation Rules
- **Mission %:** Amount sent to headquarters
- **Area %:** Amount retained at area level
- **District %:** Amount retained at district level
- **Branch %:** Amount retained at branch level
- **Total must equal 100%**

### 3. Management Options
- **Active Types:** Available for contributions
- **Inactive Types:** Disabled but preserved
- **Edit:** Modify allocations (future contributions only)
- **Delete:** Only if no contributions exist

### 4. Workflow
1. Branch admin creates type
2. Sets allocation percentages
3. Mission admin receives notification
4. Type becomes active immediately
5. Available in contribution forms

## Creating a New Type

### Step 1: Basic Information
1. Click "Create New Type"
2. Enter a descriptive name
3. Select appropriate category
4. Add optional description

### Step 2: Set Allocations
1. Enter percentage for each level
2. Ensure total equals 100%
3. System validates automatically
4. Preview allocation breakdown

### Step 3: Save
1. Review all settings
2. Click "Create Contribution Type"
3. System generates unique code
4. Mission admin notified

## Managing Types

### Edit Type
- Click edit icon on active type
- Modify allocations if needed
- Changes affect future contributions only
- Existing contributions maintain original allocations

### Deactivate Type
- Click deactivate icon
- Type becomes inactive
- Cannot be used for new contributions
- Historical data preserved

### Activate Type
- Click activate on inactive type
- Type becomes available again
- Maintains original settings

## Best Practices
1. Use clear, descriptive names
2. Follow standard allocation patterns
3. Document special purposes
4. Review regularly for relevance
5. Coordinate with mission admin
