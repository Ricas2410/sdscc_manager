# Welfare Approval Guide

## Overview
The Welfare Approval system manages requests for financial assistance, ensuring proper oversight and timely processing of welfare payments.

## Access
- **URL:** `/expenditure/welfare/requests/`
- **Permissions:** Branch Admin+, for approval
- **Menu:** Expenditure â†’ Welfare Requests

## Features

### 1. Request Management
- **List View:** All welfare requests
- **Status Tracking:** Pending, Approved, Rejected, Paid
- **Filters:** By status, branch, type
- **Search:** Find specific requests

### 2. Review Process
- **View Details:** Full request information
- **Approve:** Approve with one click
- **Reject:** Reject with reason
- **Bulk Actions:** Approve multiple requests

### 3. Request Details
- **Requester Information:** Who submitted
- **Recipient Details:** Who receives assistance
- **Amount:** Requested amount
- **Purpose:** Reason for assistance
- **Attachments:** Supporting documents

### 4. Status History
- **Created:** Initial submission
- **Approved:** Approval details
- **Rejected:** Rejection reason
- **Paid:** Payment confirmation

## Approval Workflow

### For Branch Administrators
1. Access welfare requests list
2. Review pending requests
3. Click request for details
4. Approve or reject based on policy
5. System notifies requester

### For Mission Administrators
- Can approve any branch request
- Override branch decisions
- View all welfare activities
- Generate reports

## Processing Requests

### Approve Request
1. Click approve icon
2. Confirm approval
3. System updates status
4. Notifies requester
5. Ready for payment

### Reject Request
1. Click reject icon
2. Enter rejection reason
3. System updates status
4. Notifies requester
5. Reason recorded

### Bulk Approval
1. Select multiple requests
2. Click "Approve Selected"
3. Confirm bulk action
4. All requests approved
5. Notifications sent

## Payment Processing
- **Mark as Paid:** After disbursement
- **Payment Date:** Recorded automatically
- **Payment By:** Who processed payment
- **Receipt:** Optional upload

## Reports and Analytics
- **Summary Cards:** Totals by status
- **Monthly Stats:** Approval trends
- **Branch Performance:** Comparison
- **Export Data:** Excel/CSV available

## Best Practices
1. Review requests promptly
2. Verify recipient information
3. Check for duplicate requests
4. Document decisions clearly
5. Maintain audit trail

## Policies
- **Approval Limits:** Based on role
- **Required Documentation:** Varies by amount
- **Review Timeline:** Within 3 business days
- **Appeals Process:** Through mission admin

## Notifications
- **Requester:** Notified of status changes
- **Branch Admin:** New request alerts
- **Mission Admin:** High-value requests
- **Recipient:** Payment confirmations

## Troubleshooting
- **Cannot approve:** Check permissions
- **Request not showing:** Verify filters
- **No notifications:** Check email settings

## Support
For policy questions or system issues, contact the welfare department or system administrator.
